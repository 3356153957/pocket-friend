# Pocket Friend 产品需求文档（PRD）

**版本**：v1.0  
**日期**：2026-07-23  
**状态**：MVP 设计阶段  
**目标读者**：产品、设计、开发、硬件、评委/投资人

---

## 一、一句话介绍

**Pocket Friend 是一个挂在脖子上的千禧年 8bit 风格社交硬件，把现实世界的偶遇变成可收集、可回忆、可互动的像素小人游戏。**

---

## 二、项目背景与目标

### 2.1 背景

- 当下年轻人进入 **快社交 / 泛社交** 时代：展会、咖啡馆、市集、共创空间中，人与人频繁擦肩而过，但真实连接越来越难建立。
- **OPC（One Person Company / 独立创作者）** 兴起：大量一人公司、独立开发者、自由职业者需要找到互补伙伴，但缺乏低成本的现实连接工具。
- **社交记忆碎片化**：见过很多人、加了很多联系方式，但很快忘记谁是谁、在哪里见过、为什么加。
- 现有线上社交工具（ dating app、社交平台）带来倦怠感，用户渴望现实生活中的「偶遇惊喜」。

### 2.2 产品目标

1. **降低现实社交破冰成本**：通过硬件震动 + 游戏化仪式，让偶遇自然发生。
2. **解决脸盲与记忆流失**：自动记录合影、时间、地点、共同标签，让每次连接都有上下文。
3. **创造可收藏的人脉世界**：把真实人脉转化为 8bit 像素小岛上的居民，提供情感价值和长期回访动力。
4. **打造 Adventure X 现场最抓眼球的 demo**：用项链硬件 + 捕捉仪式 + 即时家园反馈，形成强烈的视觉与情感记忆。

## 三、目标用户

### 3.1 核心用户画像

| 维度 | 描述 |
|------|------|
| 年龄 | 18-32 岁 |
| 身份 | 独立开发者、设计师、硬件创客、音乐人、自由职业者、学生创业者 |
| 场景 | 展会、黑客松、咖啡馆、公园、市集、共享办公空间 |
| 痛点 | 想认识同频的人但不好意思主动搭话；社交后忘记对方是谁；线上社交疲惫 |
| 诉求 | 低压力、高趣味、有上下文的现实连接方式 |

### 3.2 用户故事

- **作为** 一名参加 Adventure X 的硬件创客，**我希望** 在人群中快速找到也对硬件感兴趣的人，**从而** 找到潜在合伙人。
- **作为** 一个经常参加活动的自由职业者，**我希望** 每次认识的新人能自动留下合影和相遇信息，**从而** 解决脸盲和记忆混乱。
- **作为** 一个偶尔不想社交的人，**我希望** 一键让设备离线，**从而** 不被打扰。

## 四、核心体验流程

```
佩戴设备 → 在线探索 → 附近匹配 → 震动+雷达找人 → 举起设备捕捉
→ 火花+合影 → 对方进入家园 → 点击小人查看回忆
```

详细步骤：

1. 用户佩戴 Pocket Friend 项链，设备处于「在线探索」状态。
2. 当附近 10 米内出现标签匹配的人时，设备震动，越靠近震感越强，屏幕显示雷达找人 UI。
3. 双方同时举起胸前的设备对准彼此，屏幕显示「捕捉中…」。
4. 捕捉成功时，两边同步震动，屏幕火花四溅，共同标签炸成星星，设备自动拍下双人合影。
5. 对方变成用户 8bit 私人小岛上的新居民 / 家具 / NPC。
6. 用户点击小人，查看合影、联系方式、第一次见面时间/地点/共同标签。
7. 用户不想社交时，按一键离线，屏幕显示睡觉的小精灵 + 勿扰模式 icon。

## 五、功能需求（按优先级排列）

### 5.1 P0 — MVP 必须实现

P0 功能是 Adventure X demo 与产品最小可用版本的核心，必须在开发周期内完成。

| 模块 | 功能 | 需求描述 |
|------|------|----------|
| 硬件 | 项链式硬件形态 | 可穿戴项链设备，集成屏幕、摄像头、震动马达、至少一个物理按钮 |
| 硬件 | 在线/离线切换 | 一键切换在线探索与离线勿扰状态，离线时对外界不可见 |
| 硬件 | 匹配震动反馈 | 匹配成功时设备震动，当前版本可用 Mock 或简化距离算法模拟 |
| 硬件 | 捕捉仪式 | 双方举起设备触发捕捉流程，屏幕显示「捕捉中…」、火花动画、共同标签 |
| 硬件 | 自动合影 | 捕捉成功瞬间，两个设备同时拍摄双人合影并保存 |
| App | 用户标签录入 | 用户手动填写 3-5 个自由标签，如 `#硬件` `#AI` `#找合伙人` |
| App | 标签匹配 | 基于标签重合度进行匹配，当前可用预置用户数据 + Mock 场景 |
| App | 私人小岛家园 | 展示 8bit 像素风格的私人小岛，每个被捕捉的人显示为像素小人 |
| App | 小人详情页 | 点击小人查看合影、联系方式、第一次见面时间/地点/共同标签 |
| App | 离线状态显示 | App 中同步显示设备在线/离线状态与勿扰 icon |
| 系统 | Mock 数据支持 | 支持预设用户、预设标签、预设相遇场景，保证 demo 可复现 |
| 系统 | Demo 模式 | 可快速进入预设双人匹配 + 捕捉 + 家园反馈的完整流程 |

### 5.2 P1 — 重要功能，MVP 后尽快补齐

P1 功能显著提升产品完整度和 demo 表现力，应在 MVP 验证后优先实现。

| 模块 | 功能 | 需求描述 |
|------|------|----------|
| 硬件 | 屏幕雷达找人 UI | 匹配时在硬件屏幕上显示雷达圈、脉冲点、距离提示 |
| 硬件 | 离线视觉反馈 | 离线时屏幕显示睡觉的小精灵 + 勿扰模式 icon |
| App | 互补匹配 | 匹配算法同时考虑标签重合度与互补度，如 `想做硬件` + `会工业设计` |
| App | 扫描规则 | 双向扫描直接入家园；单向扫描需被扫描方确认；拒绝显示「逃走了」/「对方不想被打扰」 |
| App | 隐私设置 | 可设置「永远拒绝被扫描」；可删除小岛居民与合影 |
| App | 互访小岛 | 双方互相捕捉后，可参观对方的私人小岛 |
| App | NPC 行为系统 | 采用属性权重 + 条件触发器 + 事件池，让小人自主走动、互动、触发小剧情 |
| App | 标签 → 家园物品映射 | 共同标签生成对应家具/NPC 造型，如 `#硬件` → 共创工作台 |
| App | 数据持久化 | 合影、联系人、小岛状态本地存储，支持导出/删除 |

### 5.3 P2 — 扩展与增值功能

P2 功能用于后续版本和商业模式探索，当前阶段可不实现但保留设计空间。

| 模块 | 功能 | 需求描述 |
|------|------|----------|
| 硬件 | 真实距离感知 | 引入 UWB / 蓝牙 RSSI / 其他测距方案，实现更精确的 10 米内距离反馈 |
| 硬件 | 真实视觉识别 | 摄像头识别对方设备/人脸，用于捕捉触发 |
| App | 季节与节日事件 | 根据真实时间触发季节活动、节日剧情、限定家具 |
| App | 地理位置任务 | 到特定现实地点打卡解锁稀有家具 / 小人皮肤 |
| App | 多人小队任务 | 捕捉到的人可以组队完成共创挑战，生成专属岛屿纪念品 |
| App | OPC 专属模式 | 为独立创作者提供「找合伙人 / 找共创 / 找投资人」等定向匹配 |
| 商业 | 虚拟商品 | 售卖小岛皮肤、小人服装、限定家具等 |
| 商业 | 高级匹配 | 更精准的互补匹配、行业筛选、活动专属匹配等 Premium 功能 |

## 六、设计方案

### 6.1 硬件设计

#### 6.1.1 形态

- **项链式可穿戴设备**，主体为一个圆形/椭圆形小模块，尺寸约 5-6cm 直径，重量控制在 50g 以内。
- 正面为 **1.5-2.4 英寸 TFT 彩色屏幕**。
- 屏幕下方或侧面有一颗 **大按钮**，作为一键离线/确认键。
- 内置 **摄像头**，朝向正前方，用于捕捉合影。
- 内置 **震动马达**，提供匹配震动反馈。
- 通过挂绳/磁吸扣佩戴在胸前。

#### 6.1.2 状态与屏幕 UI

| 状态 | 屏幕表现 |
|------|----------|
| 在线待机 | 小精灵 idle 动画，偶尔眨眼或走动 |
| 匹配中 | 雷达圈 + 脉冲点 + 「附近发现同频信号！」文字 |
| 捕捉中 | 进度条 + 「捕捉中…」+ 双方标签闪烁 |
| 捕捉成功 | 火花动画 + 共同标签炸成星星 + 强烈震动 |
| 离线勿扰 | 小精灵睡着 + 月亮/勿扰 icon + 「Zzz…」 |
| 被拒绝 | 「逃走了」或 「对方不想被打扰」 |

#### 6.1.3 交互逻辑

- **短按一键按钮**：在线 ↔ 离线切换。
- **匹配中举起设备对准对方**：触发捕捉流程。
- **捕捉成功**：自动拍摄合影，数据同步到手机 App。

### 6.2 手机 App 设计

#### 6.2.1 整体风格

- **千禧年 8bit 像素风**：色彩鲜明、像素小人、复古电子宠物界面。
- 主色调：蓝、绿、黄、粉，营造轻松、治愈、 playful 的氛围。

#### 6.2.2 核心页面

| 页面 | 功能 |
|------|------|
| 首页/小岛 | 展示私人小岛，小人在岛上活动，家具和装饰分布其中 |
| 小人详情 | 点击小人后弹出：合影、昵称、联系方式、相遇时间地点标签、共同标签火花 |
| 标签设置 | 填写/修改 3-5 个自由标签 |
| 隐私设置 | 在线/离线、永远拒绝被扫描、删除数据 |
| 好友/小岛列表 | 可访问已互访的小岛 |

#### 6.2.3 家园物品生成规则

- 每个被捕捉的人会生成一个 **像素小人**。
- 根据双方共同标签生成 1-2 件 **家具/装饰**：
  - `#硬件` → 共创工作台
  - `#音乐` → 电吉他 / 唱片机
  - `#AI` → 服务器/机器人模型
  - `#找合伙人` + `#有项目` → 签约台 / 创业帐篷
- 互补标签可生成特殊 NPC 职业或互动场景。

### 6.3 NPC 行为系统设计

采用经典模拟生活游戏机制（参考《动物森友会》），不依赖大模型。

#### 6.3.1 小人属性

| 属性 | 说明 |
|------|------|
| 性格 | 外向/内向/好奇/安静等，影响行为频率 |
| 兴趣标签 | 继承自用户的标签 |
| 心情值 | 随互动事件上下波动 |
| 能量值 | 决定小人活动强度，夜间降低 |
| 社交倾向 | 决定主动靠近其他小人的概率 |

#### 6.3.2 条件触发器

- 时间：早晨、下午、夜晚触发不同行为。
- 附近小人标签：两个 #硬件 小人靠近工作台时触发「讨论 idea」事件。
- 家具类型：音乐小人靠近吉他时触发「弹吉他」事件。
- 心情/能量：低能量时小人会坐下休息或回家睡觉。

#### 6.3.3 事件池

预置大量小剧情，按权重随机抽取：

- 「两个 #硬件 小人在共创工作台旁画草图。」
- 「#音乐 小人弹吉他，吸引了一个路人小人围观。」
- 「小人翻开你们的合影，想起了 Adventure X 的那天。」
- 「一个 #AI 小人正在教另一个小人认识新工具。」

## 七、技术方案

### 7.1 硬件技术栈

| 组件 | 方案 |
|------|------|
| 主控 | ESP32 / 树莓派 Zero 2 W |
| 屏幕 | 1.8-2.4 英寸 TFT LCD，分辨率 240x320 |
| 摄像头 | OV2640 小型摄像头模块 |
| 震动 | 扁平震动马达 |
| 按钮 | 轻触开关，带 LED 背光 |
| 通信 | Wi-Fi（连接手机热点）/ 蓝牙 Low Energy |
| 供电 | 500-1000mAh 锂电池，USB-C 充电 |

### 7.2 App 技术栈

| 部分 | 方案 |
|------|------|
| 前端 | React Native / Flutter / Web（根据团队能力选择） |
| 渲染 | Canvas / PixiJS 实现 8bit 像素场景 |
| 数据存储 | 本地 SQLite / MMKV，云端可选 Supabase / Firebase |
| 图片存储 | 本地相册 + 可选云存储 |
| 通信 | WebSocket / HTTP 与硬件同步 |

### 7.3 NPC 行为系统技术栈

- 规则引擎：JavaScript/TypeScript 状态机。
- 事件池：JSON 配置 + 权重表。
- 定时器：基于真实时间的循环触发器。
- 数据模型：小人属性表、家具表、事件表、关系表。

### 7.4 当前 MVP 的 Mock 策略

- **匹配**：预置 2-5 组用户数据，demo 时手动触发匹配。
- **距离**：通过按钮或预设距离值模拟震动强度。
- **捕捉**：通过按钮确认或 NFC 靠近模拟双方对准。
- **合影**：摄像头真实拍摄，或从预设图库中选取。
- **家园**：预置岛屿模板，捕捉成功后动态插入小人/家具。

## 九、Adventure X Demo 方案

### 9.1 Demo 流程（3 分钟）

1. **开场**（30s）：主持人展示 Pocket Friend 项链，介绍一句话概念。
2. **匹配**（60s）：两位佩戴者从展台两侧走近，设备同时震动，屏幕雷达亮起。
3. **找人**（30s）：两人根据震动和屏幕提示在人群中对视、靠近。
4. **捕捉**（30s）：同时举起项链，屏幕显示「捕捉中…」，火花四溅，设备拍照。
5. **家园反馈**（30s）：打开 App，小岛上多出一个小居民和一件「共创工作台」。
6. **回忆**（30s）：点击小人，展示合影与相遇信息：「7 月 23 日，Adventure X，因 #硬件 相遇」。

### 9.2 Demo 风险控制

| 风险 | 应对 |
|------|------|
| 现场网络不稳定 | 提前配置好手机热点；关键流程本地 Mock |
| 匹配触发失败 | 准备手动触发按钮；预演多次 |
| 摄像头拍照失败 | 同时准备预设合影作为兜底 |
| 设备没电 | 准备备用设备与充电宝 |
| 观众干扰蓝牙/NFC | 控制 demo 区域，使用短距离触发 |

## 十、隐私与安全设计

### 10.1 数据最小化

- 仅收集：昵称、标签、合影、联系方式（用户自愿填写）、相遇时间地点。
- 不收集：精确地理位置轨迹、原始面部特征数据、社交关系图谱。

### 10.2 用户控制

- 一键离线：物理按钮直接切换，离线后不被匹配、不被扫描。
- 永远拒绝被扫描：全局设置，任何人都无法主动扫描用户。
- 删除机制：用户可随时删除某个小岛居民、合影、或清空全部数据。
- 互扫机制：单方扫描必须对方确认，杜绝偷拍/强制收集。

### 10.3 数据存储

- MVP 阶段：合影与联系人数据优先本地存储，减少云端隐私风险。
- 云端同步：如需要，采用端到端加密，用户拥有删除权。

## 十一、竞品差异化

| 竞品/替代品 | 差异点 |
|-------------|--------|
| Dating App（Tinder/Bumble） | Pocket Friend 不做约会，做现实偶遇 + 人脉记忆；不刷屏幕，用硬件震动触发。 |
| 微信/名片交换 | Pocket Friend 自动记录合影与相遇上下文，解决脸盲；用游戏化降低交换压力。 |
| LinkedIn | Pocket Friend 更轻、更有趣、更场景化，不是职业社交而是兴趣偶遇。 |
| 传统电子宠物/ Tamagotchi | Pocket Friend 的「宠物」是真实的人，有真实回忆和情感连接。 |

## 十二、成功指标

### 12.1 Demo 成功指标

- 现场 3 分钟 demo 流畅跑通，无关键失败。
- 观众能在一句话内理解产品价值。
- 至少 50% 观众表示「想试试」。

### 12.2 产品成功指标

- 单次活动中人均捕捉次数 ≥ 3。
- 用户 7 日回访率 ≥ 40%（打开小岛查看回忆/故事）。
- 用户主动填写完整标签率 ≥ 70%。
- 离线模式使用率合理（证明用户有控制感）。

## 十三、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 硬件开发周期长 | 高 | MVP 用现成开发板 + 3D 打印外壳，优先保证软件体验 |
| 隐私争议 | 高 | 明确数据最小化原则，物理一键离线，互扫确认机制 |
| 现场匹配不稳定 | 中 | 使用 Mock/手动触发兜底，多次预演 |
| 用户初期冷启动空岛 | 中 | 提供默认 NPC 岛民或引导任务 |
| 新鲜感褪去后留存低 | 中 | 通过 NPC 事件池、季节活动、新家具持续提供回访动力 |
