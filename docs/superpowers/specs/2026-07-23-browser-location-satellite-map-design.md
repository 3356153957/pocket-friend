# 浏览器定位与卫星地图设计

## 背景

当前网页通过 `expo-location` 的 `getCurrentPositionAsync` 只读取一次位置。浏览器可能先返回来自 Wi-Fi、基站或 IP 的粗略结果，页面随即把它显示为当前位置，用户无法判断该结果的可信程度。

当前高德地图只使用默认矢量底图，也没有底图切换入口。

## 目标

- 网页定位改为自动连续采样，优先采用误差半径最小的位置。
- 页面明确显示浏览器上报的定位精度。
- 默认使用高德卫星底图，并叠加道路与地名。
- 提供“卫星 / 标准”底图切换入口。
- 不修改原生端定位行为，不引入手动选点，不接入 JACOO。

## 定位采样

### 采样流程

1. 用户点击 GPS 后申请浏览器前台定位权限。
2. 权限允许后，调用 `Location.watchPositionAsync`，使用最高可用精度持续接收位置。
3. 每次收到位置时，比较 `coords.accuracy`；只保留精度更高（误差半径更小）的样本。
4. 满足任一条件时停止采样：
   - 最佳样本误差半径小于或等于 30 米；
   - 采样时间达到 15 秒。
5. 将最佳样本标记为 WGS84，沿用现有数据流转换为高德使用的 GCJ-02。
6. 若 15 秒内没有有效样本，显示定位失败并保持上一次有效位置。

### 页面状态

- 采样期间显示“正在提高定位精度…”。
- 有有效样本时显示“定位精度 ±N 米”。
- 最终精度大于 100 米时额外提示“当前浏览器仅提供粗略位置”。
- 浏览器没有 GPS 时，不承诺通过软件获得 GPS 级精度；页面必须如实显示精度。
- 重复点击 GPS 会停止旧订阅并开始一次新的采样，避免多个监听器同时运行。
- 页面卸载时清理定位订阅和定时器。

## 地图图层

地图加载完成后创建并复用三种高德官方图层：

- 标准底图：`AMap.createDefaultLayer()`；
- 卫星底图：`AMap.TileLayer.Satellite()`；
- 卫星路网标注：`AMap.TileLayer.RoadNet()`。

默认图层为卫星底图与路网标注。切换到标准模式时只显示标准底图；切回卫星模式时恢复卫星与路网图层。切换图层不得重建地图、丢失中心点、缩放级别、标记或当前选中人物。

地图控件区域增加一个明确的两态入口，文案显示当前可切换的目标或当前模式，并提供可访问名称与选中状态。

## 模块边界

- 新建纯函数模块处理定位样本比较、停止条件和状态文案，避免把可测试逻辑耦合到 React 组件。
- `App.tsx` 负责权限、订阅生命周期、计时器和把最佳样本写入现有附近人物状态。
- `DynamicVectorMap.web.tsx` 负责创建图层、维护底图模式和调用 `map.setLayers`。
- 原生占位地图组件不增加高德图层逻辑。

## 错误处理

- 权限拒绝：显示“定位权限未开启”。
- 浏览器定位接口报错：显示清理后的错误提示，并停止采样。
- 超时但已有样本：使用最佳样本，同时显示其真实精度。
- 超时且没有样本：显示“暂时无法获取位置，请检查浏览器定位权限”。
- 高德卫星图层创建失败：地图保持标准底图，并显示可恢复的图层错误状态。

## 测试

采用测试驱动开发：

1. 为定位采样纯函数先添加失败测试：
   - 首个有效样本成为最佳样本；
   - 更小误差半径替换旧样本；
   - 更差样本不会覆盖最佳样本；
   - 达到 30 米停止；
   - 达到 15 秒停止；
   - 粗略定位文案包含真实误差。
2. 为地图图层模式添加失败测试：
   - 默认模式为卫星；
   - 卫星模式选择卫星与路网；
   - 标准模式只选择标准底图；
   - 切换函数在两种模式间往返。
3. 运行全部单元测试、TypeScript 类型检查和 Web 生产构建。
4. 在本地浏览器验证默认卫星图、图层切换、地图缩放移动、人物标记以及定位状态文案。
