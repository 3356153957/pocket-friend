# Pocket Friend 附近交友 Demo 设计规格

- 日期：2026-07-23
- 状态：已确认，进入实现
- 范围：手机端附近交友最小闭环；网页仅作为 Expo Web 演示入口

## 1. Demo 目标

在杭州湖畔创研中心固定像素地图上，玩家可以获取自己的真实定位、看到附近模拟或在线玩家、查看距离和共同兴趣，并在进入个人设定范围时收到接近提示。Demo 重点验证定位、距离、匹配和提醒，不实现完整账号、聊天、支付与生产部署。

## 2. 场景与地图

地图中心为湖畔创研中心（礼贤路 9 号），高德坐标 `120.007986,30.293312`。固定地图以真实坐标边界为数据基础，用像素风重绘建筑、道路和活动区域，不直接复制地图瓦片。

首批地标：

- 湖畔创研中心
- 梦想小镇互联网村
- 仓南广场 T1
- 启行科创中心
- 杭师大仓前校区

地图坐标统一使用 GCJ-02；原生 GPS 和 JACOO 返回的 WGS84 坐标进入地图前必须转换。超出固定地图边界时，玩家标记停靠在边缘，并显示“地图区域外”，但保留原始距离计算。

## 3. 核心用户流程

1. 用户进入“相遇”，选择定位源并授权定位。
2. App 在固定像素地图中显示玩家位置、定位精度和数据新鲜度。
3. 用户开启“可被发现”，选择提醒范围与展示精度。
4. 在线状态层接收其他玩家的位置、兴趣和授权状态。
5. 系统先按可发现状态与距离筛选，再计算兴趣匹配结果。
6. 地图显示允许公开的模糊距离和共同兴趣；选中玩家后显示匹配理由。
7. 对方进入提醒范围且定位质量合格时，触发一次接近提醒；离开范围后才允许再次触发。
8. Demo 中可以切换模拟轨迹，单机演示完整进入、提醒和离开流程。

## 4. 定位源

所有定位源实现统一 `LocationProvider` 接口，输出 WGS84 坐标、精度、采集时间与来源。

### 4.1 原生定位

- 使用 Expo Location 获取前台定位。
- 拒绝权限、服务关闭或定位失败时展示明确状态，并允许重试或切换模拟定位。
- 正式方案仅依赖该适配器和未来经审核的后台定位方案。

### 4.2 JACOO 开发适配器

- 仅由本地 Gateway 读取 `GET /api/jacoo/location/latest`，请求凭据只存在 Gateway 运行环境。
- PF 客户端不得保存、传递或提交 JACOO API Key。
- 数据超过 2 分钟标记为“最后出现”，不得触发实时接近提醒。
- `PF_ENABLE_JACOO=false` 时路由不可用；生产环境强制关闭。
- JACOO 是开发期过渡能力，不得成为 PF 正式架构依赖。

### 4.3 模拟定位

- 内置围绕湖畔创研中心的确定性轨迹。
- 支持单步移动、自动移动和回到中心，便于演示与自动化测试。
- 界面必须明确显示“模拟定位”，防止与真实定位混淆。

## 5. 在线状态

定义 `PresenceRepository` 接口，Demo 提供两种实现：

- 内存实现：无需外部服务，可运行全部单机演示和测试。
- Supabase Realtime 实现：配置环境变量后发布/订阅玩家临时状态。

在线状态只包含 Demo 所需字段：匿名玩家 ID、显示名、像素角色、兴趣标签、可发现状态、位置、精度和更新时间。数据库配置缺失时自动降级为内存实现，不出现空白页。

## 6. 距离、隐私与提醒规则

### 6.1 定位质量

- 位置更新时间不超过 2 分钟才算实时。
- 精度误差大于用户提醒半径时，不触发接近提醒。
- 位置过期时可以显示最后出现的模糊距离，但不显示精确距离、不触发震动。

### 6.2 展示精度

用户可选择：`100m`、`500m`、`1km`、`区域`。双方可见的精度采用更保守的一方设置。

- `100m`：显示到 100 米档位，例如“约 300 米”
- `500m`：显示到 500 米档位，例如“约 1 公里内”
- `1km`：显示到 1 公里档位
- `区域`：只显示“同一区域”或“区域外”

未建立双向匹配的 Demo 玩家最多显示 `500m` 精度；选择玩家不会泄露对方精确经纬度。

### 6.3 附近与兴趣匹配

- 双方均开启可发现才进入候选集。
- 距离必须在当前用户设置的发现范围内。
- 至少一个共同兴趣才生成匹配理由。
- Demo 匹配分由共同兴趣占比和距离接近度组成，输出 0～100 的整数，并始终附带可读理由。

### 6.4 提醒

- 首次进入范围时产生 `entered` 事件，手机执行震动；Web 用视觉提示代替。
- 停留在范围内不重复提醒。
- 离开范围产生 `exited` 事件并重置状态。
- 定位过期、精度不足、任一方关闭可发现时均不触发。

## 7. 数据模型

核心类型包括：

- `GeoPoint`：纬度、经度、精度、采集时间、坐标系、来源
- `PlayerProfile`：匿名身份、显示名、角色、兴趣、发现与隐私设置
- `PresenceSnapshot`：玩家资料与最近位置
- `NearbyMatch`：距离、展示距离、共同兴趣、匹配分和理由
- `ProximityEvent`：进入或离开提醒范围及时间
- `PixelMapDefinition`：地理边界、像素画布、建筑与地标

## 8. 代码边界

- `packages/nearby-core`：纯 TypeScript 领域逻辑，无 React Native、网络或环境变量依赖。
- `apps/mobile`：Expo/React Native UI、原生定位、震动和运行时组合。
- `apps/gateway`：JACOO 只读代理及健康检查。
- 适配器依赖领域接口，领域核心不反向依赖任何平台。

## 9. 8-bit 交互基线

- 固定地图使用硬边像素块、道路、建筑轮廓和像素玩家标记。
- 面板采用直角或极小圆角、2～4 px 硬边框与阶梯阴影。
- 标题可使用像素字体，正文使用系统字体保证中文可读性。
- 触控区域不小于 44 pt；颜色不是唯一状态标识；定位和加载都有文字反馈。
- 手机优先，同时保证 Expo Web 在 375 px 和桌面宽度下可演示。

## 10. 验收标准

- 核心单元测试覆盖坐标转换、距离、投影、过期、隐私协商、附近筛选和提醒状态机。
- 无 Supabase、无 JACOO 凭据时，模拟模式仍可完成完整演示。
- 原生设备授权定位后能更新玩家位置。
- JACOO 适配器只通过 Gateway 使用，过期位置不会触发提醒。
- 固定地图能显示上述真实地标、当前玩家、至少 3 个模拟玩家和距离。
- 可切换定位源、发现范围和距离精度，并立即反映到候选列表。
- 测试、TypeScript 检查和 Web 构建通过；375 px 与桌面视口无横向溢出和遮挡。
