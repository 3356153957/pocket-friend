# Spark Connect 前端迁移设计

**日期：** 2026-07-24  
**目标分支：** `codex/dynamic-vector-map`  
**视觉来源：** `Birdy-Lin/spark-connect@22a3668`

## 目标

用 `spark-connect` 的完整用户流程替换现有 Expo Web 页面，同时保留 Pocket Friend 已有的高德动态地图、浏览器定位、附近的人匹配、环境变量和静态自动部署能力。页面品牌统一为 `Pocket Friend`。

## 用户流程

应用保留一个单页、五阶段的演示流程：

1. 欢迎页介绍 Pocket Friend 和感应挂坠。
2. 偏好问卷收集性格、兴趣和见面方式。
3. 挂坠设置选择震动方式、发现半径与免打扰。
4. 附近匹配展示真实高德地图、当前位置与附近人物。
5. 小家园展示已相遇人物的演示状态。

问卷、挂坠和小家园暂时只保存在浏览器内存中；刷新页面后重置。本次不增加数据库和服务端接口。

## 技术架构

`apps/mobile` 改为 React 19 + Vite + Tailwind CSS 4 的静态 SPA。迁移 `spark-connect` 的页面结构、文案层级、色彩、字体与交互，但不引入 TanStack Start 的 SSR 层，避免改变现有 Nginx 和 GitHub Actions 部署模型。

已有 `packages/nearby-core`、`apps/gateway` 和部署脚本保持原位。Vite 仍将网页输出到 `dist/web`，`scripts/prepare-sites-static.mjs` 再生成服务器使用的 `dist/client` 和 Worker 回退入口。

## 地图与定位

附近匹配页用现有高德地图替换 `spark-connect` 的 SVG 模拟地图：

- 默认卫星图叠加路网，支持切换标准图。
- 支持拖动、滚轮/手势缩放和地图控件。
- 自动启动浏览器高精度采样，15 秒内选择精度最好的样本。
- 采样到 30 米精度时提前结束；粗略位置明确显示实际精度。
- 定位失败时展示原因和“使用演示位置”操作。
- 点击人物标记时同步选择侧栏人物；侧栏可重新聚焦人物或当前位置。
- 所有 WGS84 浏览器坐标在进入高德地图和距离计算前转为 GCJ-02。

高德公开配置继续读取现有 `EXPO_PUBLIC_AMAP_KEY` 和 `EXPO_PUBLIC_AMAP_SECURITY_JS_CODE`。Vite 通过 `envPrefix` 显式允许这两个前缀，`.env.local` 不提交。

## 数据与隐私

演示人物继续由前端固定数据提供，距离、发现范围、可发现状态和位置精度协商继续使用 `nearby-core`。真实浏览器位置只在当前页面会话内使用，不写入持久化存储，也不上传到新服务。

## 响应式与可访问性

- 375px 手机宽度下，地图、详情、流程导航和按钮不横向溢出。
- 地图高度使用稳定的响应式约束，加载和错误状态不引起布局跳动。
- 图标按钮使用 Lucide 图标并提供 `aria-label` 与提示文字。
- 主要触控目标至少 44px，键盘焦点清晰可见。
- 在 `prefers-reduced-motion` 下关闭漂浮、脉冲和人物行走动画。
- 颜色不是选中、完成或错误状态的唯一提示。

## 错误处理

- 高德配置缺失：地图区域显示缺失变量名，不让整个页面崩溃。
- 地图 SDK 加载失败：显示可重试错误和附近人物列表。
- 卫星图层不可用：回退标准图并保留警告。
- 浏览器不支持定位或拒绝权限：保留页面流程，允许切换演示位置。

## 测试与验收

自动化测试覆盖流程状态、偏好有效性、附近人物视图模型、地图标记、图层切换和定位采样。最终验收运行完整测试、类型检查、生产构建，并用真实浏览器检查 1440x900 与 375x812 两个视口，验证全流程、地图渲染、定位错误回退及无横向溢出。

## 非目标

- 不接入图片像素化服务。
- 不恢复 Jacoo 定位。
- 不实现真实聊天、账户、持久化偏好或硬件蓝牙连接。
- 不在本次迁移中修改 `master` 或生产服务器。
