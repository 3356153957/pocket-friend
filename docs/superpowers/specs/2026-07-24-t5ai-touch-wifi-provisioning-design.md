# T5AI 触摸屏 Wi-Fi 配网设计

- 日期：2026-07-24
- 状态：已确认，等待书面规格审查
- 目标：在 Pocket Friend 的 320x480 竖屏触摸界面中完成 Wi-Fi 扫描、选网、密码输入、连接、保存和开机自动连接
- 适用范围：双 T5AI 硬件 Demo 固件，不作为量产配网与凭据安全方案

## 1. 背景与目标

当前 Pocket Friend 固件通过编译期 `PF_WIFI_SSID` 和 `PF_WIFI_PASSWORD` 连接热点。每次更换网络都需要重新构建和烧录，不适合两台设备的现场演示。

本阶段在现有触摸 UI 中加入设备端配网流程：

```text
待机页 Wi-Fi 入口
  -> 扫描附近网络
  -> 选择 SSID
  -> 输入密码
  -> 连接并验证
  -> 成功后保存到设备 KV
  -> 启动 UDP 双机发现
```

开机时优先读取设备 KV。存在凭据时自动连接；不存在凭据或自动连接失败时仍进入待机页，保持摄像头和电机等离线能力可用，由用户主动打开 Wi-Fi 设置。

## 2. 范围

### 2.1 本阶段包含

- 扫描附近 2.4GHz Wi-Fi。
- 竖屏触摸选择 SSID。
- 显示信号强度和网络是否加密。
- 使用触摸键盘输入最长 63 字符的密码。
- 密码默认隐藏，并提供显示/隐藏切换。
- 支持开放网络直接连接。
- 显示扫描、连接、成功和失败状态。
- 连接成功后将 SSID 和密码写入设备 KV。
- 开机读取 KV 并自动连接，失败时有限重试。
- 从待机页重新进入设置并更换网络。
- Wi-Fi 连接成功后启动现有 UDP 双机发现。

### 2.2 本阶段不包含

- 保存多个历史网络或自动选择多个候选网络。
- WPS、二维码、蓝牙配网或手机热点门户。
- 企业级 802.1X、隐藏 SSID 手工输入和静态 IP。
- 云端账号、远程凭据同步或涂鸦云配网。
- 对设备 KV 的额外应用层加密。
- 量产级凭据保护、恢复出厂设置和安全认证流程。

## 3. 官方代码复用策略

实现以涂鸦官方 `tuya/TuyaOpen` 仓库为参考，仓库使用 Apache License 2.0：

- `apps/tuya_t5_pocket/tuya_t5_pocket_ai/src/display/ui/wifi_scan_screen.c`：参考扫描 API、加载状态和 AP 列表创建方式。
- `apps/mimiclaw/wifi/wifi_manager.c`：参考 Wi-Fi 初始化、事件回调、有限重试、KV 读取保存和扫描结果释放方式。
- TuyaOpen 内置 LVGL `lv_keyboard` 与 `lv_textarea`：用于触摸密码输入。

不整目录复制 `tuya_t5_pocket_ai`，原因如下：

- 官方 Pocket UI 固定为 384x168 横屏，Pocket Friend 是 320x480 竖屏。
- 官方扫描页只展示 AP，没有点击选择、密码输入和连接闭环。
- 官方扫描页没有释放 `tal_wifi_all_ap_scan()` 返回的 AP 列表。
- 官方自制键盘最多输入 15 个字符，不满足 Wi-Fi 密码最长 63 字符的要求。
- 官方自制键盘会将输入文本打印到日志，不能直接用于密码。

因此只复用 API 调用方式和状态处理思路，UI、事件边界和数据所有权继续遵循现有 Pocket Friend 固件结构。

## 4. 总体架构

新增 `pf_wifi_config` 模块作为 Wi-Fi 的唯一所有者：

```text
pf_ui
  产生扫描、选网、输入密码、连接、返回等动作
        |
        v
pf_app
  串行处理动作与 Wi-Fi 事件
        |
        v
pf_wifi_config
  扫描 / 连接 / 重试 / KV / 状态
        |
        +---- 连接成功 ----> pf_transport
                              只负责 UDP
```

职责边界：

- `pf_ui`：创建页面、显示状态、产生输入动作，不直接调用 Wi-Fi 或 KV API。
- `pf_app`：将 UI 动作交给 `pf_wifi_config`，接收其事件并更新页面；连接成功后启动传输层。
- `pf_wifi_config`：唯一调用 `tal_wifi_init`、扫描、连接和 KV API 的模块。
- `pf_transport`：移除 Wi-Fi 初始化和编译期凭据依赖，只在网络已连接时创建 UDP socket、广播和收发消息。

`tal_wifi_init` 不能由 `pf_wifi_config` 和 `pf_transport` 重复调用。Wi-Fi 事件回调只更新连接状态并投递应用事件，不直接操作 LVGL 页面。

## 5. 状态模型

`pf_wifi_config` 使用以下状态：

```text
UNCONFIGURED
  -> SCANNING
  -> SELECTED
  -> CONNECTING
  -> CONNECTED

CONNECTING -> CONNECT_FAILED
CONNECTED -> DISCONNECTED -> CONNECTING
```

规则：

- KV 中没有有效 SSID 时进入 `UNCONFIGURED`。
- 自动连接和用户连接共用同一个 `CONNECTING` 状态和事件路径。
- 自动连接最多重试 3 次，重试间隔按 1 秒、2 秒、4 秒递增。
- 用户主动连接失败时不自动反复尝试，保留本次输入供用户修改或重试。
- 只有收到 `WFE_CONNECTED` 并成功获得站点 IP 后，才判定连接成功。
- 断线后允许后台有限重连；重连期间离线功能仍可使用。
- 更换网络时停止 UDP、断开旧连接，再尝试新凭据。

## 6. 触摸界面

现有 UI 增加三个页面，并在待机页右上角增加 Wi-Fi 设置图标。所有页面使用 320x480 竖屏布局，主要触摸目标不小于 48x48px。

### 6.1 Wi-Fi 列表页

- 顶栏：返回、标题、重新扫描按钮。
- 状态区：扫描中、未发现网络或扫描失败。
- 列表项：SSID、信号等级、锁图标。
- 相同 SSID 只显示一次，保留 RSSI 最强的 AP。
- 结果按 RSSI 从强到弱排列，隐藏 SSID 不显示。
- 点击加密网络进入密码页；点击开放网络直接进入连接状态。
- 重新扫描前清空旧结果并释放旧 AP 列表。

扫描属于可能阻塞的硬件操作，不能在 LVGL 绘制回调中直接执行。应用任务发起扫描，完成后通过应用事件将已复制的结果交给 UI。释放 AP 列表后，UI 不得保留指向驱动缓冲区的指针。

### 6.2 密码页

- 顶部显示所选 SSID 和返回按钮。
- 使用 `lv_textarea` 保存输入，最大长度为 63。
- 使用 LVGL `lv_keyboard`，支持小写、大写、数字和符号模式。
- 密码默认以圆点显示，可用眼睛图标切换显示状态。
- 返回列表时清空内存中的密码。
- 点击连接后复制密码到 Wi-Fi 配置模块，并立即清空 UI 输入控件。

开放网络允许空密码；加密网络不允许空密码。第一版不根据具体 WPA 类型执行更细的密码格式校验，由 Wi-Fi 驱动返回连接结果。

### 6.3 连接状态页

- 连接中：显示目标 SSID、加载状态和取消按钮。
- 成功：显示“已连接”和获得的 IP，短暂停留后返回待机页。
- 失败：显示“密码错误或网络不可用”，提供重试和重新选网。
- 失败时保留 `pf_wifi_config` 中的临时凭据供本次重试，但不写入 KV。

待机页 Wi-Fi 图标区分已连接、连接中和未连接三种状态。未配置或连接失败不会强制跳转配网页。

## 7. 数据与持久化

凭据长度：

- SSID：最多 32 字节，缓冲区为 33 字节并保证结尾 `\0`。
- 密码：最多 63 个可输入字符，缓冲区为 65 字节以兼容 TuyaOpen 的 `WIFI_PASSWD_LEN` 定义并保证结尾 `\0`。

KV 使用独立键名，例如：

```text
pf_wifi_ssid
pf_wifi_password
```

保存顺序：

1. 使用临时凭据连接。
2. 等待连接事件并确认取得 IP。
3. 写入密码键，再写入 SSID 键；SSID 作为配置完整标志。
4. 任一步写入失败时删除 SSID 键，避免下次启动读取半套配置。

读取时校验长度和结尾；数据非法则删除两个键并视为未配置。更换网络成功后覆盖旧凭据。日志允许打印 SSID、连接状态和错误码，禁止打印密码及其内容。

本设计只保证凭据不进入源码、Git、构建产物配置文本和串口日志。`tal_kv` 是否提供底层加密不在本阶段作保证。

## 8. 构建配置调整

现有同步和双固件构建脚本不再强制要求环境变量：

```text
PF_WIFI_SSID
PF_WIFI_PASSWORD
```

固件正式运行只使用设备 KV 中的运行时凭据。构建脚本不生成带真实热点信息的 `pf_demo_runtime_config.h`。设备 A/B 的差异仍由 `PF_DEVICE_ID` 等非敏感配置生成。

历史占位验证固件继续保持隔离，不允许作为可烧录产物交付。

## 9. 异常处理

- Wi-Fi 初始化失败：显示未连接状态，保留手工重试入口。
- 扫描失败：释放已有列表，显示重试按钮。
- 扫描结果为空：显示“未发现网络”，允许重新扫描。
- AP 列表过多：只保留去重排序后的最强 20 个结果，控制内存和页面长度。
- 连接调用立即失败：进入失败页，不保存凭据。
- 连接超时：10 秒后取消本次连接并进入失败页。
- KV 保存失败：保持当前网络连接，但提示“已连接，保存失败”；下次启动不会自动连接。
- UDP 启动失败：Wi-Fi 保持连接，沿用现有业务错误页和重试入口。
- 连接过程中用户取消：忽略本次连接的迟到成功/失败事件，不保存临时凭据。
- 页面退出、重连完成或应用复位时，清零临时密码缓冲区。

## 10. 测试与验证

### 10.1 自动化测试

按 TDD 顺序先补失败测试，再实现功能。测试至少覆盖：

- 无 KV、有合法 KV、KV 长度非法和半套 KV。
- SSID 去重、RSSI 排序、隐藏 SSID 过滤和最多 20 项限制。
- 开放网络空密码与加密网络空密码拒绝。
- 用户连接成功后保存，失败和超时不保存。
- 自动连接最多 3 次以及用户连接不无限重试。
- 取消连接后忽略迟到事件。
- `pf_transport` 不再引用 `PF_WIFI_SSID`、`PF_WIFI_PASSWORD` 或调用 `tal_wifi_init`。
- 构建脚本不要求或生成真实 Wi-Fi 凭据。
- 仓库受跟踪文件不包含明文 Wi-Fi 密码。

### 10.2 构建验证

- 运行现有 PowerShell 验证脚本。
- 执行 T5AI clean build。
- 执行 A/B 双固件全量构建。
- 记录两份固件的大小和哈希，并确认设备编号导致哈希不同。

### 10.3 上板验收

1. 无保存配置启动时进入待机页，点击 Wi-Fi 图标能看到附近网络。
2. 列表可滚动、重新扫描，重复 SSID 不重复显示。
3. 触摸键盘可输入大小写、数字和符号，密码默认隐藏。
4. 错误密码不会保存，失败后可以修改并重试。
5. 正确密码连接成功后返回待机页，两台设备能通过 UDP 互相发现。
6. 断电重启后无需重新输入即可自动连接。
7. 更换热点后新配置覆盖旧配置并在下次启动生效。
8. 未联网时摄像头和电机的本地功能仍可使用。
9. 串口日志和构建目录中不出现输入的 Wi-Fi 密码。
10. 连续执行 10 轮扫描、连接和重启，不出现花屏、死机、AP 列表泄漏或 Hello World 页面。

## 11. 后续演进

硬件闭环跑通后，可在不改变 `pf_app` 与 UI 事件契约的前提下增加：

- 多网络历史和优先级。
- 手机热点门户或 BLE 配网。
- 恢复出厂设置和清除凭据入口。
- 设备安全存储或应用层凭据加密。
- 网络诊断、信号质量和云端连接状态。
