# T5AI 按钮切换电机启停临时测试设计

- 日期：2026-07-23
- 状态：待书面规格审查
- 硬件：T5-BOARD_V102、35565 LCD、GC2145 摄像头、TB6612FNG、单个 5V 小型有刷直流电机
- 目标：复用板载 USER KEY，单击一次启动电机，再单击一次停止；屏幕开机自动显示摄像头画面

## 1. 范围

这是一次临时硬件联调，只验证 T5AI GPIO、TB6612FNG 和电机的控制链路。

本次包含：

- 开机自动显示摄像头实时画面；
- 初始化 GPIO P6、P7；
- 单击 USER KEY 在“正转”和“停止”之间切换；
- 串口输出电机状态；
- 编译、烧录和实机验证。

本次不包含：

- 反转；
- PWM 调速；
- 电机控制抽象层；
- 电流检测、堵转检测和持久化状态；
- 产品级电源、EMC 和保护设计。

## 2. 接线前提

TB6612FNG 使用 A 通道：

```text
T5AI P11-12 5V  -> VM
T5AI P11-14 3V3 -> VCC、STBY、PWMA
T5AI P11-9  P6  -> AIN1
T5AI P11-7  P7  -> AIN2
T5AI P11-13 GND -> GND
TB6612 AO1/AO2  -> 电机两根线
```

未使用的 `BIN1`、`BIN2`、`PWMB` 接 GND。电机只允许短时间空载测试；如果屏幕闪烁、花屏或开发板重启，立即断电。

## 3. 实现方案

为了保持临时测试最简单，直接修改 TuyaOpen 官方示例：

```text
examples/graphics/lvgl_camera/src/example_lvgl_camera.c
```

不新增 `motor_control.c/.h`。

增加以下应用状态：

```text
motor_running = false
```

增加三个最小函数：

- `motor_init()`：把 P6、P7 配置为推挽输出，并先写低电平；
- `motor_start()`：写入 P6=高、P7=低；
- `motor_stop()`：写入 P6=低、P7=低。

现有单击按钮回调不再切换摄像头/LVGL，而是切换 `motor_running`：

```text
停止 -> 启动正转
运行 -> 停止
```

摄像头显示状态在初始化阶段直接设为启用，使开机后自动进入实时预览。

## 4. 初始化顺序与失效安全

初始化顺序：

```text
日志与系统服务
-> 板级硬件注册
-> 电机 GPIO 初始化并强制停止
-> 按钮初始化
-> LVGL 初始化
-> 摄像头初始化
-> 摄像头显示初始化
-> 自动启用摄像头预览
```

安全约束：

- GPIO 初始化前，软件状态为“停止”；
- 初始化失败时不执行启动命令；
- 每次停止都同时把 P6、P7 拉低；
- 设备重启后不会恢复之前的运行状态；
- 不在按钮中断处理函数里做延时或阻塞操作；
- 串口日志明确输出 `[motor] started` 或 `[motor] stopped`。

## 5. 测试与验收

自动化验证采用红绿循环：

1. 先增加一个检查脚本，要求源文件包含 P6/P7 定义、开机停止、按钮启停切换和自动摄像头预览；在生产代码不存在时确认测试失败。
2. 写最小生产代码后再次运行，确认测试通过。
3. 运行已有 T5AI 板型配置测试。
4. 对 `lvgl_camera` 执行全量清理和完整构建。

实机验收：

1. 烧录后电机默认不转。
2. 屏幕自动显示摄像头画面。
3. 第一次单击 USER KEY，电机正转，日志输出 `[motor] started`。
4. 第二次单击 USER KEY，电机停止，日志输出 `[motor] stopped`。
5. 连续切换三次，无开发板重启、屏幕花屏或摄像头停止。
