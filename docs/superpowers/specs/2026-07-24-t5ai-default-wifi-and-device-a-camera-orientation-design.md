# T5AI 默认 Wi-Fi 与 A 板相机方向设计

- 日期：2026-07-24
- 状态：等待书面规格审查
- 范围：Pocket Friend 双 T5AI 演示固件

## 1. 目标

新固件在设备 KV 中没有用户保存的网络时，自动连接现场默认 Wi-Fi。A 板的摄像头预览和拍照结果旋转 180°，B 板保持现有方向。

默认 Wi-Fi 的 SSID 为 `ADVX-Players`，安全类型为 WPA2-个人。密码不进入受 Git 跟踪的文件、不写入日志，也不写入知识库；它只在本机同步固件时注入生成的运行时头文件。最终二进制必然包含联网所需凭据，因此该方案仅用于现场演示固件，不作为量产凭据方案。

## 2. Wi-Fi 行为

启动连接优先级固定为：

1. 设备 KV 中存在触摸屏保存的 SSID 和密码时，连接已保存网络。
2. KV 不存在或内容无效，且构建时提供了默认网络时，连接默认网络。
3. 两者都不存在时，保持离线并允许用户进入触摸屏 Wi-Fi 设置页。

默认网络连接成功后不写入 KV。这样用户在触摸屏选择其他网络并连接成功后，新配置会保存到 KV，并在以后启动时覆盖默认网络。默认连接和已保存网络都沿用现有三次自动重试；失败后不阻塞摄像头、电机和触摸界面。

WPA2 类型不新增枚举或分支，由 `tal_wifi_station_connect()` 根据非空密码与接入点协商。日志只允许输出连接状态，不输出密码。

## 3. 凭据注入

`sync-lvgl-camera.ps1` 从本机环境读取默认网络参数，验证 SSID、WPA2 密码长度并转义为 C 字符串，然后写入 `D:\TuyaOpen\examples\graphics\lvgl_camera\include\pf_demo_runtime_config.h`：

```c
#define PF_DEFAULT_WIFI_ENABLED 1
#define PF_DEFAULT_WIFI_SSID "runtime_ssid"
#define PF_DEFAULT_WIFI_PASSWORD "runtime_password"
```

未提供参数时生成禁用状态和空占位值，使公开仓库仍可在无凭据环境构建。验证脚本扫描 Git 跟踪文件，确保没有默认密码明文。

## 4. A 板相机方向

根因是 A/B 固件共用 LCD、相机驱动和 UI 数据路径，但 A 板摄像头模组物理安装方向相反。GC2145 的当前 TDL 接口未暴露镜像或旋转参数，修改 LCD 旋转会同时破坏整个界面和触摸坐标。

采用设备级 RGB565 旋转：

- 同步脚本为 A 板生成 `PF_CAMERA_ROTATION_180 1`，B 板生成 `0`。
- YUV422 转换为预览 RGB565 后，A 板原地交换首尾像素，实现 180° 旋转，无需第二块 480x480 帧缓冲。
- JPEG 解码为拍照结果 RGB565 后执行同一变换。
- B 板编译为无操作，不改变画面方向。

备选方案未采用：修改 GC2145 私有寄存器会侵入 TuyaOpen 驱动且缺少公开接口；旋转整个 LVGL 显示会影响全部页面和触摸；只做垂直镜像不能修正物理倒置造成的左右方向。

## 5. 测试与验收

自动化验证按 TDD 执行：

1. 先扩展固件合同测试，验证 KV 优先、默认网络兜底、凭据不落库、A/B 方向宏和预览/拍照共用旋转逻辑；确认修改前测试失败。
2. 实现最小代码后，运行固件合同、相机配置和按钮电机回归测试。
3. 分别同步 A/B 配置，确认生成头文件中 A 为 180°、B 为 0°，且默认网络只存在于未跟踪的 TuyaOpen 构建目录。
4. 全量 clean/build A 板固件，检查构建成功和镜像产物。
5. 烧录 A 板后验证：无 KV 时自动连接 `ADVX-Players`；打开 Camera 后画面方向正确；拍照结果方向正确；触摸界面方向和坐标不变。

如果实机显示只是上下镜像而非整幅倒置，则停止继续叠加修改，重新以单一方向变换验证硬件表现。
