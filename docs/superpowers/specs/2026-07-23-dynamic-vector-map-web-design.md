# Pocket Friend 动态矢量地图 Web 设计规格

- 日期：2026-07-23
- 状态：已确认，等待书面规格审查
- 范围：远程服务器上的 Expo Web Demo
- 不包含：iOS/Android 原生动态地图、像素头像转换、地图瓦片自建

## 1. 目标

将当前固定尺寸、手工拼接的卫星瓦片替换为真正可交互的高德矢量地图。地图必须像常规地图一样支持平移、缩放、旋转和倾斜，并与 Pocket Friend 的定位、附近人物、匹配和隐私状态联动。

本次优先保证静态导出的 Expo Web Demo 部署到远程服务器后可直接使用。原生手机地图留到后续阶段，避免在当前 Web Demo 中引入未使用的原生依赖和开发构建要求。

## 2. 技术方案

Web 地图使用高德地图 JavaScript API 2.0，并通过 `@amap/amap-jsapi-loader` 加载。现有定位和人物数据已经统一为 GCJ-02，能够直接交给高德地图，无需增加另一套坐标偏移。

新增 Web 专用地图组件，建议位置：

- `apps/mobile/src/map/DynamicVectorMap.web.tsx`
- `apps/mobile/src/map/mapModel.ts`
- `apps/mobile/src/map/mapConfig.ts`

`DynamicVectorMap.web.tsx` 负责高德实例、覆盖物和地图事件；纯 TypeScript 文件负责配置校验、人物标记数据和地图指令，便于使用现有 `node:test` 测试。

Expo 页面只向地图组件传递：

- 当前定位来源与位置
- 可见人物的 GCJ-02 坐标、角色颜色和选中状态
- 当前选中的人物 ID
- 地图聚焦请求
- 人物标记点击回调

地图组件不得反向依赖匹配、隐私或 Gateway 实现。

## 3. 凭据与安全边界

Web 构建使用以下公开环境变量：

- `EXPO_PUBLIC_AMAP_KEY`
- `EXPO_PUBLIC_AMAP_SECURITY_JS_CODE`

浏览器必须获取这些值才能加载高德地图，因此它们不能被视为服务端秘密，也不得宣传为“隐藏在服务器”。实际保护措施是在高德控制台设置 Web 端允许域名，并分别配置开发和正式域名。

变量只写入部署环境和本地未跟踪的 `.env`；仓库中的 `.env.example` 只包含变量名和说明，不包含真实值。

## 4. 数据流与定位规则

```text
SIM / 浏览器 GPS
        ↓
坐标标准化为 GCJ-02
        ↓
createNearbyGameState
        ↓
人物位置、匹配结果、公开距离
        ↓
DynamicVectorMap.web.tsx
        ↓
高德矢量底图与人物覆盖物
```

SIM 模式从湖畔轨迹起步并逐步移动。GPS 模式使用浏览器返回的真实位置，转换为 GCJ-02 后放置当前人物标记。

当前实现中“无论 GPS 坐标为何都把自己固定在湖畔”的行为必须移除。地图初始视图位于湖畔，但真实定位成功后人物标记必须反映真实位置。

距离计算、共同兴趣、隐私精度和附近候选仍由 `nearby-core` 决定。地图内部可以使用坐标放置标记，但界面不得直接显示其他人物的原始经纬度。

## 5. 地图交互

地图支持：

- 鼠标拖动平移
- 滚轮和触摸板缩放
- 双指触屏缩放
- 旋转和倾斜
- `+`、`−` 缩放按钮
- “回到湖畔”
- “定位到我”
- “一键朝北”

地图初次加载时展示湖畔和已有可见人物。GPS 首次成功后可以自动定位到用户；后续位置变化只更新标记，不持续抢占地图中心。

用户手动移动地图后，SIM 轨迹更新不得自动把地图拉回湖畔。

点击地图人物标记时：

1. 高亮对应标记。
2. 更新页面选中的人物。
3. 展示既有匹配详情。

点击附近人物列表时：

1. 更新选中人物。
2. 地图平滑移动到该人物。
3. 不改变当前缩放级别，除非人物当前不在可视区域。

## 6. 视觉与响应式

高德负责真实道路、建筑、文字和地图交互。Pocket Friend 继续使用高对比 8-bit 面板和像素人物标记，避免把整张地图伪装成像素画。

地图覆盖物需包含：

- 当前人物与其他人物的像素角色标记
- 选中态边框或光环
- 当前定位来源 `SIM/GPS`
- 当前缩放级别
- 加载、离线和错误状态

布局基线：

- 375 px 手机 Web：地图高度约 420 px
- 桌面 Web：地图高度约 560 px
- 地图宽度始终跟随内容容器，不产生横向滚动
- 地图控制按钮的点击区域不小于 44×44 px
- 颜色不是人物选中状态的唯一提示

## 7. 生命周期与性能

高德实例只在容器和配置就绪后创建一次。React 状态变化时更新标记和视图，不重复创建地图。

组件卸载时必须：

- 移除地图事件监听
- 移除人物覆盖物
- 销毁地图实例

人物标记更新使用 ID 对比，复用已有覆盖物；不存在的人物标记被移除，新人物才创建覆盖物。单次 SIM 更新不得重新加载地图脚本或重建底图。

## 8. 错误和降级

### 8.1 配置缺失

未设置 Key 或安全密钥时，不调用高德 Loader。地图区域显示配置说明和缺失变量名，附近列表及其他 Demo 功能继续可用。

### 8.2 地图加载失败

域名白名单、安全密钥、网络或高德脚本错误统一进入可重试状态。错误信息不得包含完整凭据。

### 8.3 GPS 失败

权限拒绝、服务关闭或读取超时后保留当前地图视图，显示明确提示，并允许用户切回 SIM。

### 8.4 无效人物坐标

非有限数值或超出合法经纬度范围的人物不会生成覆盖物。单个人物数据错误不能使整张地图白屏。

### 8.5 离线

保留最后一次附近列表和状态文字。地图区域显示离线或加载失败状态，不伪造新的位置。

## 9. 测试策略

按 TDD 实现以下纯逻辑：

1. 高德配置完整、缺失和空白值判断。
2. 有效 GCJ-02 人物位置转换为地图标记模型。
3. 无效经纬度被过滤。
4. 当前人物和选中人物生成正确的可访问标签和选中态。
5. 点击列表人物生成地图聚焦指令。
6. GPS 坐标不再被强制替换为湖畔固定坐标。

移除旧的 `satelliteTiles` 生产代码和测试，避免继续维护静态瓦片拼接。

现有坐标转换、距离、隐私、匹配、提醒、定位 Provider 和 Gateway 测试必须继续通过。

## 10. 浏览器验收

在真实浏览器中验证：

- 375×812 和桌面视口无横向溢出或遮挡
- 鼠标拖动、滚轮缩放、按钮缩放可用
- 旋转、倾斜和一键朝北可用
- 回到湖畔和定位到我可用
- SIM 移动只更新标记，不抢回地图中心
- 地图标记和附近列表双向联动
- Key 缺失时显示降级界面而不是白屏
- 控制台没有未处理错误

交付前必须重新运行：

```text
npm test
npm run typecheck
npm run build:web
```

## 11. 完成标准

- Web Demo 使用动态高德矢量地图，不再拼接卫星图片
- 地图可以平移、缩放、旋转和倾斜
- 当前人物和附近人物按真实 GCJ-02 坐标显示
- SIM、GPS、人物选择和隐私逻辑与地图联动
- 配置缺失和加载失败均有可恢复状态
- 自动化测试、类型检查、Web 构建和真实浏览器验收全部通过
