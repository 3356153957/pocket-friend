# Pocket Friend 项目总待办

> 本文件用于逐项跟踪 Pocket Friend 的产品验证、设计、开发、测试、硬件和网页工作。完成一项并满足对应验收标准后，再进入下一项。

**目标：** 先验证脸盲辅助与附近社交的真实需求，再完成手机 App MVP，最后验证球形挂件和辅助网页端。

**范围说明：** 这是跨子系统的总路线图，不直接规定代码结构。进入开发阶段后，脸盲辅助、附近社交、定位安全、挂件和网页端分别建立带具体文件路径与测试命令的实现计划。

**设计依据：** `docs/superpowers/specs/2026-07-23-pocket-friend-design.md`

## 使用方法

1. 每次只推进一个编号事项。
2. 完成事项中的全部要求后，把 `- [ ]` 改成 `- [x]`。
3. 将调研记录、原型、测试报告或技术文档保存到仓库对应目录。
4. 每完成一段可独立使用的成果，立即提交并推送。
5. 未通过阶段关卡时，不提前投入下一阶段的大规模开发或硬件量产。

## 阶段一：产品方向

- [x] **1. 明确产品定位**

  完成标准：确认产品名为 Pocket Friend，手机 App 为主，球形挂件为可选外设，网页端为辅。

- [x] **2. 明确两条独立主线**

  完成标准：脸盲辅助与附近社交可以独立使用，并且只在用户授权时关联。

- [x] **3. 完成项目设计大纲**

  完成标准：产品功能、隐私边界、挂件、网页端、AI、MVP 和测试重点均已形成书面规格。

## 阶段二：需求验证

- [ ] **4. 确定首轮封闭测试场景**

  完成标准：选择一个具体场景、一个城市和 20～30 名潜在用户。推荐首轮选择线下兴趣活动或展会，因为参与者有明确社交意愿，也存在记忆新认识对象的需求。

- [ ] **5. 编写用户访谈提纲**

  完成标准：分别覆盖脸盲经历、人物记录习惯、线下交友需求、定位接受度、震动提醒、拍照授权和挂件使用意愿；每次访谈控制在 30～45 分钟。

- [ ] **6. 招募首批访谈对象**

  完成标准：招募 10～15 人，其中至少 5 人经常记不住姓名或面孔，至少 5 人有线下拓展社交圈的需求；记录基本背景但不收集无关敏感信息。

- [ ] **7. 完成用户访谈**

  完成标准：每位受访者形成一份匿名记录，包含真实经历、现有解决方法、最大顾虑和使用意愿，不用引导性问题替用户确认需求。

- [ ] **8. 整理需求验证结论**

  完成标准：输出两条主线各自的高频痛点、使用场景、放弃使用原因和最重要的三个功能；明确哪些假设得到支持，哪些假设被否定。

- [ ] **9. 制定 MVP 成功指标**

  完成标准：为人物卡复用率、像素头像记忆帮助、匹配查看率、双向互选率、接近提醒误报率、关闭率和线下见面转化率定义计算方式与首轮目标。

### 阶段关卡 A

只有当访谈证明至少一条主线存在稳定需求，并且用户能够接受当前授权规则时，才进入完整交互设计。若两条主线的核心假设均未得到支持，应先修改产品定位并重新验证。

## 阶段三：App 产品设计

- [ ] **10. 完成 App 信息架构**

  完成标准：确定“相遇、人物、消息、我的”四个一级入口及全部 MVP 页面，说明每个页面的职责和入口。

- [ ] **11. 完成脸盲辅助用户流程**

  完成标准：覆盖拍照或导入、照片授权、像素化、人物卡编辑、搜索、见面前复习、本地保存、云备份、恢复和删除。

- [ ] **12. 完成附近社交用户流程**

  完成标准：覆盖建档、开启可发现、AI 推荐、区域筛选、距离提醒、双向互选、聊天、见面、解除匹配、屏蔽和举报。

- [ ] **13. 制定权限与距离矩阵**

  完成标准：逐项列出拍照、人物卡云备份、可发现状态、模糊距离、精确距离和实时位置的开启条件、双方可见内容、有效时间与撤回结果。

- [ ] **14. 绘制低保真交互原型**

  完成标准：两条主线的核心流程均可从头走到尾；空状态、拒绝授权、定位失败、生成失败、挂件断连和举报入口均有明确界面。

- [ ] **15. 进行第一轮可用性测试**

  完成标准：至少 5 名未参与设计的用户独立完成创建人物卡、查找人物、开启附近匹配、调整距离权限和屏蔽用户五项任务；记录完成率、耗时和困惑点。

- [ ] **16. 修改并冻结 App MVP 需求**

  完成标准：修正可用性问题，形成 MVP 页面清单、功能清单、状态清单和验收标准；删除没有验证价值的附加功能。

### 阶段关卡 B

只有当核心任务可用性测试通过、权限规则没有明显误解，并且 MVP 范围可以由一个小团队完成时，才进入技术开发。

## 阶段四：技术设计

- [ ] **17. 确定技术栈**

  完成标准：确定移动端框架、服务端语言与框架、数据库、对象存储、地图定位、即时消息、推送、AI 服务和部署平台，并记录选择理由、成本和替代方案。

- [ ] **18. 拆分子系统实现计划**

  完成标准：分别为脸盲辅助、附近社交、定位与权限、消息与安全编写实现计划；每份计划包含具体文件路径、接口、失败测试、最小实现、验证命令和提交步骤。

- [ ] **19. 设计数据模型与接口**

  完成标准：定义账号、人物卡、像素头像、社交档案、匹配、授权、距离级别、会话、举报和设备的数据结构；明确人物私有数据不能进入公共推荐。

- [ ] **20. 完成安全与隐私评审**

  完成标准：检查未注册对象数据、照片处理、精确位置、授权撤回、账号删除、日志保留、加密、异常查询和骚扰防护；记录风险等级与具体控制措施。

## 阶段五：手机 App MVP 开发

- [ ] **21. 开发脸盲辅助最小闭环**

  完成标准：用户可以拍照或导入、生成像素头像、创建人物卡、搜索人物、查看见面摘要，并通过自动化测试验证核心数据流程。

- [ ] **22. 开发人物卡加密云备份**

  完成标准：支持备份、恢复、导出和删除；未注册对象的人物卡只对创建者可见，不进入公共搜索、匹配或人脸索引。

- [ ] **23. 开发附近社交最小闭环**

  完成标准：用户可以建立兴趣档案、获得带理由的推荐、开启可发现、查看模糊距离、互选和解除匹配，并有自动化测试覆盖主要状态转换。

- [ ] **24. 开发定位与接近提醒**

  完成标准：区域定位完成候选筛选，蓝牙辅助判断接近；双方选择不同精度时采用更保守设置；定位漂移或精度不足时不触发精确提醒。

- [ ] **25. 开发消息与安全功能**

  完成标准：实现匹配后聊天、限时位置共享、隐身、屏蔽、举报和撤回授权；安全功能不受付费限制。

- [ ] **26. 完成双主线授权关联**

  完成标准：社交匹配对象可以加入人物库，已有卡片可在双方授权后关联账号；解除关联后停止资料与位置同步，但保留用户自己的私人笔记。

- [ ] **27. 完成 App 系统测试**

  完成标准：通过自动化测试、真机测试、弱网测试、后台耗电测试、定位误差测试、通知到达测试和授权撤回测试；所有阻断级缺陷已关闭。

- [ ] **28. 发布封闭测试版**

  完成标准：向阶段二确定的 20～30 名用户发布，提供反馈、举报和紧急停止定位入口，并建立测试期间的问题响应流程。

- [ ] **29. 评估封闭测试结果**

  完成标准：按照第 9 项指标输出报告，决定保留、修改或删除的功能，并明确是否进入扩大测试和挂件开发。

### 阶段关卡 C

只有当 App 能证明接近提醒、人物记忆或两者之间至少一个闭环产生持续价值，并且没有无法控制的定位安全问题时，才投入完整挂件开发。

## 阶段六：球形挂件

- [ ] **30. 编写挂件产品需求书**

  完成标准：明确原创球形外观、佩戴位置、按键动作、震动模式、状态灯、蓝牙范围、续航目标、充电方式、重量、防护等级和丢失解绑机制。

- [ ] **31. 制作无摄像头交互原型**

  完成标准：先验证按键、震动、状态灯、蓝牙连接、佩戴舒适度和误触，不因摄像头增加首轮硬件验证成本。

- [ ] **32. 测试挂件基础体验**

  完成标准：至少 10 人完成佩戴、配对、兴趣确认、震动识别和丢失解绑测试；记录连接成功率、续航、误触和佩戴问题。

- [ ] **33. 制作带摄像头原型**

  完成标准：实现快捷拍照、强制点亮且不可关闭的拍照提示灯、加密传输和传输后清除；挂件不能读取历史联系人数据。

- [ ] **34. 完成小批量硬件测试**

  完成标准：完成可靠性、跌落、温升、续航、充电、弱连接、隐私提示和手机兼容性测试，再决定是否量产。

## 阶段七：网页端与扩大测试

- [ ] **35. 开发基础官网**

  完成标准：包含产品说明、App 下载、隐私政策、帮助中心、硬件信息和联系方式。

- [ ] **36. 开发用户管理网页**

  完成标准：支持设备管理、远程解绑、人物卡备份管理、数据导出、授权查看和账号注销；网页默认不持续获取定位。

- [ ] **37. 开发审核与运营后台**

  完成标准：支持举报处理、申诉、风险账号限制、设备状态和官方社交区域管理，并保留必要审计记录。

- [ ] **38. 开展扩大测试**

  完成标准：在多个场景中验证用户留存、人物卡复用、互选、提醒误报、安全事件和挂件使用情况，并与首轮指标比较。

- [ ] **39. 制定正式发布计划**

  完成标准：明确发布范围、应用商店材料、隐私合规文件、客服与举报响应、监控告警、数据备份、事故预案和版本回滚方案。
