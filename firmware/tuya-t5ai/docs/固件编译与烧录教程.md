# T5AI 固件编译与烧录教程

本文用于教别人给 Pocket Friend 的两块涂鸦 T5AI-Board 开发板编译和烧录固件。当前固件基于 TuyaOpen 的 `examples/graphics/lvgl_camera` 示例，通过本仓库的覆盖层同步到 `D:\TuyaOpen` 后编译。

## 适用对象

- 开发板：涂鸦 `T5AI-Board / T5-BOARD_V102`
- LCD：`T5-BOARD-35565LCD`
- 摄像头：GC2145
- 固件分支：`codex/tuya-t5ai-firmware`
- SDK 根目录：`D:\TuyaOpen`

## A 板和 B 板的区别

同一套源码会根据构建参数生成 A/B 两个固件：

| 项目 | A 板 | B 板 |
| --- | --- | --- |
| `PF_DEVICE_ID` | `A` | `B` |
| `PF_PEER_ID` | `B` | `A` |
| 摄像头旋转 | `PF_CAMERA_ROTATION_180 1` | `PF_CAMERA_ROTATION_180 0` |
| 协议角色 | 发起 capture prepare / capture / success | 响应 prepare ack / capture |

因此烧录前必须确认自己正在构建的是 A 还是 B，不能把同一个 bin 同时刷进两块板。

```powershell

D:\TuyaOpen\tools\tyutool\tyutool_cli.exe
```

打开 PowerShell，进入固件分支工作区：

```powershell
cd D:\Users\17\Documents\friend\.worktrees\tuya-t5ai-firmware
```

每次编译前，先在当前 PowerShell 进程补齐 TuyaOpen 需要的环境变量：

```powershell
$env:OPEN_SDK_ROOT = "D:\TuyaOpen"
$env:OPEN_SDK_PYTHON = "D:\TuyaOpen\.venv\Scripts\python.exe"
$env:OPEN_SDK_MAKE_BIN = "D:\TuyaOpen\.tools\make\4.4.1"
$env:OPEN_SDK_MAKE = "D:\TuyaOpen\.tools\make\4.4.1\make.exe"
$env:VIRTUAL_ENV = "D:\TuyaOpen\.venv"
$env:PYTHONIOENCODING = "utf-8"
$env:PATH = "D:\TuyaOpen\.venv\Scripts;D:\TuyaOpen\.tools\make\4.4.1;D:\TuyaOpen;" + $env:PATH
```

如果需要让固件内置默认 Wi-Fi 或启用服务器心跳/照片上传，在同步源码前额外设置下面这些环境变量。不要把真实 token、密码写进 Git、教程或聊天记录。

```powershell
$env:PF_DEFAULT_WIFI_SSID = "<WIFI_SSID>"
$env:PF_DEFAULT_WIFI_PASSWORD = "<WIFI_PASSWORD>"
$env:PF_ADMIN_HOST = "<SERVER_HOST_OR_LAN_IP>"
$env:PF_ADMIN_PORT = "4311"
$env:PF_DEVICE_HEARTBEAT_TOKEN = "<DEVICE_TOKEN>"
```

不设置这些变量也可以编译，只是固件不会内置默认 Wi-Fi，也不会启用服务器心跳和照片上传。板子仍可使用已保存的 Wi-Fi 配置。

## 编译 B 板固件

先同步覆盖层到 TuyaOpen，并指定 `-DeviceId B`：

```powershell
powershell -ExecutionPolicy Bypass -File firmware\tuya-t5ai\scripts\sync-lvgl-camera.ps1 -TuyaOpenRoot D:\TuyaOpen -DeviceId B
```

检查同步输出。若没有设置服务器变量，会看到类似：

```text
WARNING: Server heartbeat disabled: set PF_ADMIN_HOST and PF_DEVICE_HEARTBEAT_TOKEN before building.
PASS: lvgl_camera dual-demo overlay synced for device B (default Wi-Fi: False; server heartbeat: False).
```

核对运行时头，确认是 B 板：

```powershell
Get-Content -Raw D:\TuyaOpen\examples\graphics\lvgl_camera\include\pf_demo_runtime_config.h
```

关键内容应包含：

```c
#define PF_DEVICE_ID 'B'
#define PF_PEER_ID 'A'
#define PF_CAMERA_ROTATION_180 0
```

然后编译：

```powershell
cd D:\TuyaOpen\examples\graphics\lvgl_camera
D:\TuyaOpen\.venv\Scripts\python.exe D:\TuyaOpen\tos.py clean -f
D:\TuyaOpen\.venv\Scripts\python.exe D:\TuyaOpen\tos.py build
```

成功时输出里会出现：

```text
====================[ BUILD SUCCESS ]===================
Target    : lvgl_camera_QIO_1.0.0.bin
Platform  : T5AI
Board     : TUYA_T5AI_BOARD
```

产物路径：

```powershell
D:\TuyaOpen\examples\graphics\lvgl_camera\dist\lvgl_camera_1.0.0\lvgl_camera_QIO_1.0.0.bin
```

建议复制成清晰文件名：

```powershell
New-Item -ItemType Directory -Path D:\TuyaOpen\artifacts\pocket-friend-demo -Force | Out-Null
Copy-Item -LiteralPath D:\TuyaOpen\examples\graphics\lvgl_camera\dist\lvgl_camera_1.0.0\lvgl_camera_QIO_1.0.0.bin -Destination D:\TuyaOpen\artifacts\pocket-friend-demo\device-b.bin -Force
Get-FileHash -Algorithm SHA256 -LiteralPath D:\TuyaOpen\artifacts\pocket-friend-demo\device-b.bin
```

## 烧录固件

烧录时建议只插一块板，避免把 A/B 刷反。

先列串口：

```powershell
D:\TuyaOpen\tools\tyutool\tyutool_cli.exe list-ports --plain
```

常见输出类似：

```text
COM4    1a86:55d2    0    -    wch.cn USB-Enhanced-SERIAL-A CH342 (COM4)
COM5    1a86:55d2    2    -    wch.cn USB-Enhanced-SERIAL-B CH342 (COM5)
```

优先选择 `USB-Enhanced-SERIAL-A` 那个端口烧录，例如 `COM4`。

烧录 B 板：

```powershell
D:\TuyaOpen\tools\tyutool\tyutool_cli.exe write -d t5ai -p COM4 -b 921600 -f D:\TuyaOpen\artifacts\pocket-friend-demo\device-b.bin --plain
```

烧录 A 板：

```powershell
D:\TuyaOpen\tools\tyutool\tyutool_cli.exe write -d t5ai -p COM4 -b 921600 -f D:\TuyaOpen\artifacts\pocket-friend-demo\device-a.bin --plain
```

烧录成功通常会看到：

```text
Handshake OK
Write OK
Reboot OK
Flash OK
```

## 烧录后验证

打开串口日志：

```powershell
D:\TuyaOpen\tools\tyutool\tyutool_cli.exe monitor -d t5ai -p COM4 --plain
```

启动日志里应能看到设备身份：

```text
Pocket Friend dual-device demo: B
Platform board: TUYA_T5AI_BOARD
```

或：

```text
Pocket Friend dual-device demo: A
Platform board: TUYA_T5AI_BOARD
```

两块板同时上电后，应能互相发现。若只发现对方但不继续确认，当前固件会回到 idle，不应再误报 `Capture failed`。双方确认并进入同步倒计时后，固件会预热相机 encoded stream，再抓取 JPEG。

## 推荐教学流程

1. 只插 B 板。
2. 运行 `list-ports`，确认 `USB-Enhanced-SERIAL-A` 端口。
3. 同步 `-DeviceId B`。
4. 核对 `pf_demo_runtime_config.h` 里的 `PF_DEVICE_ID 'B'`。
5. 运行 `tos.py clean -f` 和 `tos.py build`。
6. 复制固件为 `device-b.bin`。
7. 用 tyutool 烧录 B 板。
8. 用 monitor 确认日志自报 `B`。
9. 拔掉 B 板，插 A 板，重复同样流程但使用 `-DeviceId A` 和 `device-a.bin`。
10. 两块板都刷完后同时上电，测试互相发现、确认、倒计时和同步拍照。

## 常见问题

### `make: command not found`

当前 PowerShell 没有把 TuyaOpen 的 make 放进 PATH。重新执行：

```powershell
$env:PATH = "D:\TuyaOpen\.venv\Scripts;D:\TuyaOpen\.tools\make\4.4.1;D:\TuyaOpen;" + $env:PATH
```

### `Python was not found`

平台 bash 调用了裸 `python`，但 PATH 里没有 TuyaOpen 虚拟环境。重新执行环境变量准备步骤。

### `PF_SERVER_HEARTBEAT_ENABLED 0`

这是因为同步时没有设置 `PF_ADMIN_HOST` 和 `PF_DEVICE_HEARTBEAT_TOKEN`。如果需要照片上传，必须在同步前设置服务器环境变量。

### 看到多个 COM 口，不知道刷哪个

CH342 会枚举出 A/B 两个串口。通常使用 `USB-Enhanced-SERIAL-A` 那个端口烧录。教学时最好一次只插一块板。

### 串口被占用或拒绝访问

可能有残留 monitor 占着串口。先查看并结束 tyutool：

```powershell
Get-Process | Where-Object { $_.ProcessName -like "*tyutool*" }
Stop-Process -Id <PID> -Force
```

### reset 后串口短暂消失

这是板子重启或重新枚举。等几秒后重新运行：

```powershell
D:\TuyaOpen\tools\tyutool\tyutool_cli.exe list-ports --plain
```

### 刷完不确定是 A 还是 B

不要靠文件名猜。用 monitor 看启动日志里的：

```text
Pocket Friend dual-device demo: A
```

或：

```text
Pocket Friend dual-device demo: B
```
